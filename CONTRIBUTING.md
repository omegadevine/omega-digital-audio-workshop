# Contributing to Omega Digital Audio Workshop

Thank you for your interest in contributing to the Omega Digital Audio Workshop!

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Code Standards

- Follow C++17 standards
- Use consistent naming conventions (camelCase for variables, PascalCase for classes)
- Comment complex algorithms and audio processing logic
- Write clean, maintainable code
- Test your changes thoroughly

## Reporting Bugs

Please include:
- Operating system and version
- Audio hardware details
- Steps to reproduce
- Expected vs actual behavior
- Any error messages or logs

## Feature Requests

We welcome feature suggestions! Please:
- Check existing issues first
- Provide detailed use cases
- Explain how it benefits users

## Audio Processing Guidelines

- Minimize latency in real-time processing
- Handle buffer sizes efficiently
- Implement proper error handling for audio devices
- Document DSP algorithms clearly

## Code Review Process

All submissions require review. We use GitHub pull requests for this purpose.

Thank you for helping make Omega DAW better!
